# Brainstorming Lab

This folder owns detailed idea development before an idea becomes a real execution repo.

## Role

```text
planning-everything-track = priorities, capacity, status, locators, durable lessons
brainstorming-lab = detailed brainstorms, assumptions, options, critique, graduation rationale
standalone project repo = executable work, scripts, experiments, outputs, manuscript assets
```

## Routing Rule

When a brainstorm becomes larger than a daily-note scratchpad, put the detailed idea note under `ideas/`.

When it becomes executable:

1. Keep the detailed idea rationale here.
2. Create or update the standalone execution repo.
3. Update the planning repo only with locator, status, capacity, and next-action links.
