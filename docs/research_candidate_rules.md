# Research Candidate Rules

Research candidates are generated from clusters, not isolated enthusiasm.

## Promotion Gates

A cluster becomes a research candidate only when all gates pass:

- cluster size is at least 2
- average maturity score is at least 65/100
- average testability score is at least 10/15
- average evidence score is at least 8/15
- average feasibility score is at least 8/15
- a normalized problem statement exists
- every source idea has a baseline
- every source idea has metrics

## Readiness Score

| Dimension | Points |
| --- | ---: |
| Problem importance | 20 |
| Scientific testability | 20 |
| Evidence depth | 15 |
| Method feasibility | 15 |
| Novelty contribution | 15 |
| Execution path | 15 |

## Levels

| Score | Level |
| --- | --- |
| 0-49/100 | `topic_only` |
| 50-64/100 | `discussable` |
| 65-79/100 | `small_experiment_ready` |
| 80-89/100 | `proposal_ready` |
| 90-100/100 | `independent_repo_or_paper_ready` |

Research-ready does not mean validated, deployed, clinically safe, legally
confirmed, or production-ready. It means the cluster has enough structure to
support a bounded experiment or proposal.
