# Idea Lifecycle

## First Principle

An idea has a lifecycle, and each lifecycle state needs one canonical home.

This repo owns the thinking stage:

- what the idea is
- why it might matter
- which assumptions might fail
- which options are plausible
- what would make it real or kill it

It does not own daily capacity or final execution.

When the idea becomes real, create a graduation packet and move execution into a new standalone project repo.

## States

| State | Meaning | Default action |
| --- | --- | --- |
| `spark` | Captured but not developed | Keep light or create an idea file |
| `active` | Worth thinking about now | Run bounded sessions |
| `parked` | Useful later, not now | Add review trigger |
| `killed` | Not worth continuing | Record reason |
| `ready-to-graduate` | Has a real next output | Create graduation packet |
| `graduated` | Moved to standalone repo | Link to project repo |

## Graduation Criteria

An idea can become a project only when it has:

- a clear user, owner, or audience
- a first concrete output
- a next physical action
- evidence that it is not only a mood or curiosity
- a boundary for what not to build
- a repo name and canonical home

## Stop Conditions

Kill or park the idea when:

- the core assumption fails
- the timing conflicts with current capacity
- the idea only creates more maintenance
- it duplicates an existing repo
- it has no action after two sessions

## Review Questions

- Did this idea change future action?
- Does it deserve a project repo?
- Does it only need a durable lesson?
- Is the next test small enough?
- What should planning know, and what should stay here?
