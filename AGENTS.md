# AGENTS.md

This folder is the local brainstorming lab for ideas that are too detailed for daily planning notes but not yet ready to become execution repos.

## Role

- Own detailed brainstorms, critique, assumptions, options, and project-graduation rationale.
- Keep planning-facing summaries short and link back to the canonical idea note.
- Do not turn every idea into an implementation repo.

## Boundaries

Planning repo:

```text
../planning-everything-track/
```

Owns:

- priority
- capacity
- daily note bridge lines
- weekly review
- project locators
- durable learned notes

Brainstorming lab owns:

- detailed idea development
- raw thinking organized into sections
- critique / objections
- possible research directions
- keep / drop / park decisions

Execution repos own:

- code
- scripts
- experiments
- manuscripts
- deliverable artifacts

## Routing Rule

If a brainstorm becomes longer than a few daily-note bullets, create or update an `ideas/*.md` file here.

If the idea becomes executable:

1. Keep the rationale and decision trail here.
2. Create or update the standalone execution repo.
3. Update the planning repo with only locator, status, capacity impact, and next action.

## Safety Note

For security or AI-safety brainstorms, keep the notes defensive and research-oriented.

Prefer:

- threat models
- benchmark design
- governance implications
- safe explanations
- non-operational placeholders

Avoid:

- optimized attack prompts
- step-by-step harmful instructions
- credential, exploit, or evasion details
